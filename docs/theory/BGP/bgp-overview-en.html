BGP (Border Gateway Protocol) Overview

1. BGP Characteristics

  • Unlike OSPF and EIGRP, BGP doesn't implement its own transport protocol
  • Uses TCP as its transport protocol (TCP port 179)

2. BGP Session Establishment

Two-Step Process:

  • Step 1: TCP Session Establishment (TCP Three-way Handshake)
  • Step 2: BGP Peering Session Establishment

3. Router Operation

  • When BGP is enabled, router listens on TCP port 179
  • Router acts as both TCP Client and Server
  • In simultaneous SYN case, router with higher BGP ID becomes Client

4. BGP Peer Types

  • iBGP (Internal BGP): When ASNs match
  • eBGP (External BGP): When ASNs differ

5. BGP Implementation

  • Primary protocol for Service Provider Networks
  • Used in enterprises with multiple ISP connections (Multi-homing)
  • Core protocol of Internet Backbone routing

Key Points to Remember:

  • BGP operates on TCP port 179
  • Requires Neighbor Configuration on both sides
  • Ideal for large networks requiring detailed route control
  • Supports policy-based routing and route filtering
